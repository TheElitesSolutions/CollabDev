# 08 – Project Timeline (Gantt Overview)

The development of CollabDev+ is divided into six main phases.
Each phase produces a concrete, demo-able outcome aligned with the project goals.

| Phase | Timeframe         | Main Activities                                                                                                   | Key Deliverables                                                      |
|-------|-------------------|-------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------|
| A     | Week 1 – Week 2   | - Set up repos (frontend & backend)  <br> - Configure PostgreSQL, Prisma, Redis **and Neo4j connection**  <br> - Create basic Prisma schema (users, projects, members)  <br> - Implement Auth (login)  <br> - Seed demo users & one demo project  <br> - Basic `/workspace/[projectId]` page | Working login + ability to open a project workspace; all databases (PostgreSQL, Redis, Neo4j) reachable from backend |
| B     | Week 3 – Week 4   | - Design DB models for task columns & tasks  <br> - Implement Kanban APIs (`/board`, task CRUD)  <br> - Build Kanban UI in Next.js  <br> - Integrate drag-and-drop  <br> - Add WebSocket updates for task changes | Real-time Kanban board (create/move tasks across columns)             |
| C     | Week 5 – Week 6   | - Design `files` model and seed demo code project  <br> - Implement file tree + file content APIs  <br> - Integrate Monaco editor in frontend  <br> - Add WebSocket gateway for collaborative edits  <br> - Handle cursors & basic conflict resolution | Collaborative IDE where multiple users edit the same file             |
| D     | Week 7 – Week 8   | - Design `pages` model & `structure` JSON  <br> - Implement page CRUD APIs  <br> - Implement code generation from page structure  <br> - Build visual website builder UI (canvas + components + properties)  <br> - Link pages to generated code files in IDE | Visual website builder synced to an underlying code file              |
| E     | Week 9 – Week 10  | - Implement `messages` model & chat history API  <br> - Add chat WebSocket events and UI panel  <br> - Implement WebSocket-based WebRTC signaling (offers/answers/ICE)  <br> - Integrate simple 1:1 video call in workspace | Integrated project chat + working 1:1 video call inside the workspace |
| F     | Week 11 – Week 12 | - Implement `artifact_links` model & REST APIs  <br> - Connect tasks ↔ files ↔ pages in the UI (linking panels)  <br> - **Sync artifact links to Neo4j via GraphModule**  <br> - (Optional) Implement a simple “graph of related items” view using Neo4j queries  <br> - UX polishing, bug fixing, performance tweaks  <br> - Prepare final demo scenario & documentation | Artifact linking between code/tasks/pages, backed by Neo4j graph; polished MVP ready for presentation |
